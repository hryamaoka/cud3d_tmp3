# Cub3D プロジェクト要件 (日本語訳)

注意: このドキュメントは一般的な `cub3d.pdf` (バージョン2) の要件に基づいています。詳細は最新のオリジナルのPDFを参照してください。

## 概要
Cub3Dの目標は、レイキャスティング（Ray-Casting）技術を使用して、Wolfenstein 3D風の3Dグラフィックエンジンを作成することです。

## 必須要件 (Mandatory part)

### 1. グラフィックスライブラリ
*   **MiniLibX** を使用すること。
*   OSに標準搭載されているウィンドウシステム（API）や他のライブラリを使用しても良いが、C言語で書かれていなければならない。
*   OpenGlやmlilibx以外のグラフィックライブラリの使用は禁止。

### 2. 一般要件
*   全てのファイルは `Wall`, `Wextra`, `Werror` フラグを付けてコンパイルすること。
*   メモリリークは一切許容されない。
*   `Makefile` を作成すること（`all`, `clean`, `fclean`, `re`, `bonus` ルールを含む）。
*   グローバル変数は使用禁止。

### 3. 表示機能
*   迷路の中をプレイヤーの視点（一人称視点）で表示すること。
*   ウィンドウ管理（別のウィンドウへの切り替え、最小化など）がスムーズに行えること。
*   壁の種類（東西南北）に応じて異なるテクスチャを表示すること。
*   床と天井の色を個別に設定できること。

### 4. 操作機能
*   キーボード操作で視点移動（左右回転）と移動（AD/矢印キーで左右、WS/矢印キーで前後）ができること。
*   `ESC` キーを押すか、ウィンドウの「×」ボタン（赤バツ）を押すと、プログラムが正しく終了すること。

### 5. パース（.cubファイル）
プログラムは `.cub` 形式の設定ファイルを引数として受け取る必要があります。

#### マップ要素
*   **テクスチャ**: `NO`, `SO`, `WE`, `EA` に続きパスを指定。
*   **色**: `F` (床), `C` (天井) に続きRGB値 (0-255) を指定。

#### マップ構造
*   `0`: 何もない空間
*   `1`: 壁
*   `N`, `S`, `E`, `W`: プレイヤーの初期位置と向き
*   マップは壁 (`1`) で完全に閉じられていること。
*   マップ内にスペース (` `) があっても良いが、マップの一部として扱われる。

### 例
```
NO ./path_to_the_north_texture
SO ./path_to_the_south_texture
WE ./path_to_the_west_texture
EA ./path_to_the_east_texture

F 220,100,0
C 225,30,0

111111
100101
101001
111111
```

## ボーナス要件 (Bonus part)
必須要件が完璧な場合のみ評価されます。
*   壁の衝突判定（壁を通り抜けられない）。
*   ミニマップシステム。
*   ドアの開閉。
*   アニメーション付きスプライト。
*   マウスによる視点操作（左右）。
*   しゃがむ、ジャンプなどの追加アクション。
*   ライフバー、スコア、環境音など。
