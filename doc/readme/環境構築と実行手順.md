# セットアップと実行ガイド

本プロジェクトのビルド手順およびテスト実行コマンド（正常系・異常系・リークチェック）について記載します。

## 1. 環境構築 (Setup)

リポジトリのクローンと、サブモジュール（MiniLibX等の依存ライブラリ）の初期化を行います。

```bash
git clone https://github.com/hryamaoka/cub3d_tmp.git
cd cub3d_tmp
git submodule update --init --recursive
```

## 2. 必須パート (Mandatory Part)

### コンパイル
```bash
make
```

### 実行確認

#### 正常系テスト
有効なマップファイルを指定して実行します。
```bash
./cub3D assets/maps/valid/test.cub
```

#### 異常系テスト
プレイヤーが存在しない無効なマップなどの挙動を確認します。
```bash
./cub3D assets/maps/valid/no_player.cub
```
*(期待値: "Error" メッセージが表示され、正しく終了すること)*

## 3. ボーナスパート (Bonus Part)

ボーナス機能（衝突判定、ミニマップ等）を含むコンパイルと実行確認です。
※本プロジェクトでは `make` でボーナス機能も含まれる構成になっている場合は `make` で十分ですが、明示的に分かれている場合は `make bonus` 等を使用してください（現状のMakefile設定に準拠）。

### コンパイル
```bash
make
```

### 実行確認

#### 正常系テスト
ボーナス機能確認用のマップを使用します。
```bash
./cub3D assets/maps/valid/bonus.cub
```

#### 異常系テスト
壁が閉じていないマップでの挙動を確認します。
```bash
./cub3D assets/maps/valid/not_closed_bonus.cub
```
*(期待値: "Error" メッセージが表示され、正しく終了すること)*

## 4. メモリリークチェック (Leak Check)

Valgrindを使用してメモリリークがないかを確認します。

```bash
valgrind -s --track-origins=yes --leak-check=full --show-leak-kinds=all ./cub3D assets/maps/valid/bonus.cub
```
