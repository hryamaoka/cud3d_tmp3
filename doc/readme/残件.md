# 残件リスト (Remaining Tasks)

プロジェクト完了および提出に向けて、今後対応が必要な項目をまとめました。

## 1. ソースコードの改善
*   **コードのブラッシュアップ**: 不要なコメントの削除、Norminette対応（42のコーディング規約遵守）。
*   **内容理解の深化**: 各モジュール（特にレンダリング計算部分）の詳細な動作を完全に理解し、説明できるようにする。

## 2. Makefileの修正 (提出用最終調整)
現状の開発用設定から、提出用の形式へ変更する必要があります。

1.  **`-g` フラグの削除**
    *   デバッグ情報を削除し、最適化されたコンパイル設定にする。
    
2.  **ソースファイルの明記 (ワイルドカード `*` の廃止)**
    *   現状: `$(shell find ... "*.c")` などで自動取得している。
    *   変更: `SRCS` 変数に、全ての `.c` ファイルパスを1つずつ明示的に記述する形式に変更する。
    *   *理由: 評価環境によっては `find` コマンドやワイルドカード展開が期待通りに動作しないリスクを避けるため。*

3.  **`clean` ルールの修正**
    *   以下の行を削除する。
        ```makefile
        find srcs -name "*.o" -delete
        ```
    *   *理由: 標準的な `rm $(OBJS)` のみに戻し、Makefileの記述をシンプルかつ移植性の高いものにする。*
