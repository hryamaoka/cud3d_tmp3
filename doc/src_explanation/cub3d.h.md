# `include/cub3d.h` 詳細解説

## 概要
プロジェクト全体で共有される主要な構造体、定数、関数プロトタイプを定義するメインヘッダーファイルです。全ての主要な`.c`ファイルからインクルードされます。

## コード詳細解説

| 行番号 | コード | 解説 |
| :--- | :--- | :--- |
| **27-32** | `typedef struct s_tex` | **テクスチャパスマネジメント構造体**<br>設定ファイルから読み込んだ4方向(NO, SO, WE, EA)のテクスチャファイルパスを保持します。 |
| **34-38** | `typedef struct s_color` | **RGB色構造体**<br>R, G, B 各チャンネルの整数値(0-255)を保持します。天井と床の色設定に使用されます。 |
| **40-44** | `typedef struct s_map` | **マップデータ構造体**<br>二次元配列 `grid` と、その幅`width`・高さ`height`を管理します。 |
| **46-54** | `typedef struct s_player` | **プレイヤー情報構造体**<br>位置(`x`, `y`)、方向ベクトル(`dir_x`, `dir_y`)、カメラ平面ベクトル(`plane_x`, `plane_y`)を倍精度浮動小数点数で保持します。 |
| **56-74** | `typedef struct s_ray` | **レイキャスティング計算用構造体**<br>各フレームのレイ計算時に使用される一時変数群です。DDAアルゴリズム用の変数(`side_dist`, `delta_dist`, `step`)や、最終的な壁までの距離(`perp_wall_dist`)が含まれます。 |
| **76-84** | `typedef struct s_img` | **MLX画像構造体**<br>MiniLibXの画像オブジェクトを管理します。`img`(ポインタ)、`addr`(ピクセルデータ先頭)、および画像メタデータを含みます。 |
| **86-89** | `typedef struct s_sprite` | **スプライト構造体**<br>マップ上のスプライトオブジェクトの位置情報(`x`, `y`)を保持します。 |
| **91-113** | `typedef struct s_game` | **メインゲーム構造体**<br>アプリケーション全体の状態を保持する巨大な構造体です。MLXインスタンス、画像バッファ、パースされた設定（テクスチャ、色、マップ）、プレイヤー状態、Ｚバッファ(`z_buffer`)などを一元管理します。関数間で状態を受け渡す主要なコンテキストとなります。 |
| **115-138** | `void init_game(...)` ... | **関数プロトタイプ**<br>各モジュール（初期化、パース、コア、レンダリング）の公開関数のプロトタイプ宣言です。ここに宣言することで、他のファイルから呼び出し可能になります。 |
