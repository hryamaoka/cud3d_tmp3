# `src/core/game_init.c` 詳細解説

## 概要
ゲーム構造体 `t_game` の初期化を行うファイルです。未定義動作やメモリゴミによるバグを防ぐため、パース処理やウィンドウ作成が始まる前に、全てのメンバ変数を安全な初期値（0, NULL, または -1）に設定します。

## 依存関係
*   `cub3d.h`: `t_game` 構造体。
*   `init.h`: 関連する初期化関数定義。

## 関数詳細

### `init_game`
#### プロトタイプ
`void init_game(t_game *game)`

#### 戻り値
なし

#### 解説
`t_game` 構造体全体をゼロクリア (`ft_bzero`) し、その後でパース前の初期状態として `-1` が必要なメンバ変数を設定します。
*   `ft_bzero` により、全てのポインタ変数は `NULL` に、整数変数は `0` に初期化されます。
*   その後、設定ファイル等で上書きされるべき変数（床・天井の色、マップサイズ、プレイヤー位置など）に `-1` を代入することで、パース後に「値が正しく設定されたか」を検証可能にします。

### コード行別解説

| 行番号 | コード | 詳細解説 |
| :--- | :--- | :--- |
| 16 | `ft_bzero(game, sizeof(t_game));` | **ゼロクリア**<br>構造体の全メモリ領域を0で埋めます。これにより、冗長な `NULL` 代入や `0` 初期化を省略でき、コードを簡潔に保てます。 |
| 17-30 | `game->floor.r = -1;` ... | **未設定値の初期化**<br>`0` が有効な値となり得る変数（RGB値など）や、必須項目（マップサイズ、プレイヤー位置）について、初期値を `-1` に設定します。これにより、パース処理で値が更新されなかった場合のエラーチェックが可能になります。 |
